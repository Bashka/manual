====== Переменные ======

  * site
    * baseurl - корневой URL адрес сайта
    * source (''./'') - адрес каталога, содержащего исдохные файлы
    * destination (''./_site'') - адрес каталога, в который будут компилироваться исходные файлы
    * plugins (''./_plugins'') - адрес каталога, содержащего классы плагинов
    * layouts (''./_layouts'') - адрес каталога, содержащего шаблонные файлы
    * data_source (''./_data'') - адрес каталога, содержащего файлы данных
    * defaults - умолчания
    * data - данные из файлов данных
    * pages - страницы
    * posts - посты блога
    * коллекция - документы данной коллекции

<code html>
{% for album in site.albums %}
  <li>{{ album.output }}
{% endfor %}
</code>

    * collections - массив пользовательских коллекций

<code>
collections:
  - my_collectionA
  - my_collectionB
</code>

    * categories.категория - коллекции данной категории
    * tags.тег - коллекции с данным тегом
  * content - содержимое страницы (используется в файлах layout)
  * page
    * layout - адрес файла, используемого в качестве шаблона страницы (''null'', если следует не использовать шаблон)

Файл ''_layouts/default.html''
<code html>
<html>
  <head>...</head>
  <body>
    ...
    {{ content }}
    ...
  </body>
</html>
</code>

    * permalink - постоянная ссылка на страницу
    * published - следует ли компилировать файл
    * categories - категории коллекции
    * tags - теги коллекции
  * коллекция
    * label - название коллекции
    * docs - массив документов
    * files - массив статичных файлов
    * output - следует ли рендерить файлы коллекции
    * relative_directory - путь к каталогу документов коллекции относительно корня исходников сайта
    * directory - полный путь к каталогу документов коллекции
  * документ коллекции
    * collection - название коллекции
    * path - пусть к документу относительно каталога коллекции
    * name - базовое имя документа
    * title - название документа
    * output_ext - расширение итогового файла
    * content - неотрендеренный документ
    * output - отрендеренный документ
    * url - URL документа

====== Конфигурация ======

Переменные могут быть заданы в файле ''_config.yml''. Значения этих переменных вступят в силу только при повторном запуске сервера.

====== Умолчания ======

Переменная ''site.defaults'' используется для задания переменных типа ''page'' в качестве значений по умолчанию.

<code>
defaults:
  -
    scope: # Условие, при котором применяется данное умолчание странице
      path: "/projects" # Адрес расположения целевых страниц (пустая строка для всех страниц проекта)
      type: "pages" # Имя коллекции, к которой должна относиться страница
    values: # Применяемые переменные
      layout: "default"
</code>

====== Файлы данных ======

Переменные могут быть перечислены в файлах данных, располагаемых по умолчанию в каталоге ''_data'' в формате YAML, JSON или CSV. Переменные из этих файлов доступны в переменной ''site.data''.

Файл ''_data/authors.yml'':

<code>
- name: Foo Bar
  github: foobar
...
</code>

<code html>
<ul>
  {% for author in site.data.authors %}
    <li>
      <a href="https://github.com/{{ author.github }}">{{ author.name }}</a>
    </li>
  {% endfor %}
</ul>
</code>

====== Вводные ======

Вводные это переменные страницы, устанавливаемые в начале файла.

<code html>
---
# Вводные
variable: data
...
---
<div>
  ...
</div>
</code>
