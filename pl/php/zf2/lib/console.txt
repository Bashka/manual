Пакет [[http://framework.zend.com/manual/current/en/modules/zend.console.routes.html|Console]] позволяет выполнить роутинг запросов к системе из консоли. Конфигурация этого пакета налогична пакету [[:ЯП:ZF2:lib:Mvc:Router]].

<code php>
[
  'router' => [
    'routes' => [
      // HTTP роутинг
    ]
  ],
  // Роутинг консоли
  'console' => [
    'router' => [
      'routes' => [
        'my-first-route' => [
          'type'    => 'simple', // Данный тип роутинга используется по умолчанию, потому это свойство может быть опущено
          'options' => [
            'route'    => 'foo bar', // Использует команду вида "php index.php foo bar"
            'defaults' => [
              'controller' => 'Application\Controller\Index',
              'action'     => 'password'
            ]
          ]
        ]
      ]
    ]
  ],
]
</code>

В качестве компонентов роутинга (''options.route'') могут использоваться следующие маркеры:

  * **Литералы** - эти маркеры должны присутствовать в команде;

<code>
foo (a|b) [c|d] - foo за которым следует a или b за которыми может следовать c или d
</code>

  * **Флаги** - эти маркеры могут изменять поведение команды;

<code>
foo [--bar|-b] - foo за которым может следовать --bar или -b
</code>

  * **Именованные параметры** - эти маркеры определяют данные для команды. Получить значение этих маркеров можно с помощью команды ''$this->getRequest()->getParam('маркер')'';

<code>
foo [<userId>|--user=] - foo за которым можно указать идентификатор пользователя или --user=идентификатор
</code>

  * **Группы параметров** - эти маркеры аналогичны флагам, но используют иной синтаксис. Получить значение этих маркеров можно с помощью команды ''$this->getRequest()->getParam('маркер')''.

<code>
foo [--bar|-b]:myFlag - foo за которым следует --bar или -b
</code>
