====== Структура редактора ======
Скрипты, используемые для работы редактора Vim делятся на следующие (//первичные//) группы по области действия:
  * Глобальные скрипты - файлы скриптов, расположенные в //глобальном каталоге редактора//. Действие этих скриптов распространяется на всех пользователей системы. Адрес этого каталога можно получить из переменной среды ''VIMRUNTIME'';
  * Пользовательские скрипты - файлы скриптов, расположенные в //пользовательских каталогах редактора//. Действие этих скриптов распространяется на того пользователя, в домашнем каталоге которого они расположены. Адрес этого каталога отличается для различных операционных систем. Так, в ОС семейства Unix он находится по адресу ''$HOME/.vim'', а в ОС семейства Windows по адресу ''$HOME/vimfiles'';
  * Проектные скрипты - файлы скриптов, расположенные в //проектном каталоге редактора//. Действие этих скриптов распространяется на конкретный каталог (проект). Расположение проектного каталога зависит от конкретной реализации модели проектных скриптов, но как правило он расположен в каталоге самого проекта ''Proj/.vim''.

Каждая из перечисленных выше групп (каталогов) может включать в себя следующие типы (//вторичных//) подгрупп (подкаталогов):
  * ''after'' - скрипты, выполняемые после загрузки редактора;
  * ''autoload'' - скрипты, используемые для реализации [[..:struct:namespace|областей видимости]]; 
  * ''colors'' - скрипты, используемые для реализации цветовых схем (тем оформления) редактора;
  * ''doc'' - скрипты документации редактора и плагинов;
  * ''ftplugin'' - скрипты, привязанные к конкретному типу файла;
  * ''indent'' - скрипты, реализующие логику отступов для различных типов файлов;
  * ''keymap'' - скрипты, определяющие клавиатурные комбинации для различных языковых групп;
  * ''plugin'' - скрипты плагинов;
  * ''spell'' - справочники, используемые для проверки орфографии;
  * ''syntax'' - скрипты, реализующие правила синтаксической подсветки различных типов файлов.
Сюда так же входят //базовые файлы инициализации//, название которых зависит от операционной системы. Так для систем семейства Unix это файлы ''.vimrc'', а для систем семейства Windows файлы ''_vimrc''.

====== Порядок загрузки ======
Инициализация (загрузка) редактора Vim выполняется путем последовательного исполнения всех файлов инициализации вторичных подгрупп для каждой первичной группы.

Схема инициализации редактора:
  - Глобальный каталог редактора:
    - Базовый файл инициализации;
    - Файлы в каталоге ''plugin'';
    - Файлы в оставшихся каталогах вторичной группы, кроме каталога ''after''.
  - Пользовательский каталог редактора:
    - Базовый файл инициализации;
    - Файлы в каталоге ''plugin'';
    - Файлы в оставшихся каталогах вторичной группы, кроме каталога ''after''.
  - Проектный каталог редактора:
    - Базовый файл инициализации;
    - Файлы в каталоге ''plugin'';
    - Файлы в оставшихся каталогах вторичной группы, кроме каталога ''after''.
  - Файлы каталога ''after'' из глобального каталога редактора;
  - Файлы каталога ''after'' из пользовательского каталога редактора;
  - Файлы каталога ''after'' из проектного каталога редактора.

Порядок загрузки файлов основных групп определен опцией редактора ''runtimepath'', в которой перечислены (через запятую) адреса каталогов этих групп. 

Пример содержимого опции ''runtimepath'':
<code>
/usr/share/vim/vim73,/usr/share/vim/vimfiles,/home/artur/.vim,.vim
</code>

Просмотреть ход загрузки скриптов редактора при его инициализации можно с помощью команды '':scriptnames''. Она перечисляет адреса всех скриптов, загруженных на момент ее вызова, в порядке их исполнения.

====== Структура плагинов ======
Плагин для редактора Vim это набор скриптов, которые выполняются при инициализации и добавляют новую функциональность. Конкретный плагин может включать скрипты вторичных групп, но обычно скрипты плагина относятся к каталогу ''plugin''. Эти скрипты выполняются "как есть" и используют команды и привязки редактора для функционирования.

====== Установка плагина ======
Для установки плагина необходимо:
  - Получить файлы плагина. Обычно они распространяются в виде архива;
  - Переместить файлы плагина в соответствующие каталоги вторичной группы. На пример файлы плагина из каталога ''autoload'' необходимо переместить в одноименный каталог. В зависимости от того, какой каталог основной группы выбран (глобальный, пользовательский или проектный), плагин будет действовать на уровне этой группы;
  - Выполнить дополнительные конфигурации плагина (согласно инструкции по установке) в одном из основных файлов инициализации выбранной основной группы.

====== Удаление плагина ======
Для удаления плагина необходимо:
  - Удалить все установленные ранее файлы плагина;
  - Удалить команды настройки плагина из основного файла инициализации.
