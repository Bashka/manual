====== Режимы ======

- normal - нормальный режим, позволяющий перемещаться по документу
- insert (i) - режим вставки, позволяющий добавлять данные в документ
- replace (R) - режим замены, позволяющий заменять данные в документе
- visual (v) - режим выделения, позволяющий выделять данные в документе
- visual-line (V) - режим строчного выделения, позволяющий выделять данные в документе построчно
- visual-block (^v)- режим блочного выделения, позволяющий выделять даннные в документе блочно
- ex (:) - командный режим, позволяющий выполнять консольные команды редактора
- ex-package (Q) - пакетный командный режим, позволяющий выполнять пакеты консольных команд редактора
- ex-shell (:!) - командный режим оболочки, позволяющий выполнять консольные команды оболочки
- diff - режим просмотра различий, позволяющий визуализировать различия в документах

Функция ''mode()'' возвращает маркер текущего режима в текущем буфере.
Функция ''visualmode()'' возвращает маркет последнего использованного режима выделения (v, V, ^V) в текущем буфере.

====== Запуск ======

Запустить редактор можно с помощью команды оболочки ''vim'' тремя способами:
- ''vim'' - создается пустой, безымянный буфер. Данный буфер является текущим
- ''vim файл'' - создается буфер, связанный с указанным файлом и заполненный данными из него. Данный буфер является текущим
- ''vim файлА файлБ ...'' - создается по одному буферу для каждого из перечисленных файлов, но текущим буфером будет тот, который связан с первым указанным файлом (файлА)

Открытие группы файлов:
<code bash>
vim file1 file2 file3
</code>

====== Ключи ======
- ''vim'' - открыть редактор в нормальном режиме
- ''vim -'' - открыть редактор в нормальном режиме и прочитать данные из stdin в текущий буфер
- ''vim -e'' - открыть редактор в пакетном командном режиме и прочитать команды из stdin. После выполнения пакета команд, редактор закрывается
- ''vim -d'' - открыть редактор в режиме просмотра различий 
- ''vim -R'' - открыть редактор в режиме "только чтение"
- ''vim -m'' - открыть редактор в режиме "запрет записи" (в отличии от режима "только чтение", данный режим предупреждает только при попытке записи буфера)
- ''vim -M'' - открыть редактор в режиме "запрет изменений" (в отличии от режима "запрет записи", данный режим предупреждает только при попытке изменения содержимого буфера)
- ''vim --noplugin'' - открыть редактор без загрузки плагинов
- ''vim -u rcфайл'' (устанавливается опция compatible) - использовать команды из файла для инициализации (другие rc файлы использованы не будут)
- ''vim -u NORC'' (устанавливается опция compatible) - открыть редактор без загрузки rc файлов
- ''vim -u NONE'' (устанавливается опция compatible) - открыть редактор без загрузки rc файлов и плагинов
- ''vim -D'' - запуск в режиме отладки (n - следующая команда, c - до следующей точки останова)
- ''vim -n'' - запуск редактора без использования своп-файлов
- ''vim -x'' - запуск редактора в режиме шифрования

Открытие редактора в нормальном режиме и запись вывода команды оболочки ''ls'' в текущий буфер:
<code bash>
ls | vim -
</code>

Выполнение команд из файла ''commands.vim'' с помощью интерпретатора редактора:
<code bash>
vim -e < commands.vim
</code>

====== Инициализация ======

Процесс инициализации редактора зависит от используемых ключей (''-u'', ''--noplugin''), опции ''runtimepath'', и как правило состоит из:
^ #  ^ Файл                        ^ Контекст ^ Если ^ Описание ^
| -  | rtp=$HOMEVIM,$VIM           | -------- | - | - |
| 1  | $VIM/vimrc                  | Нет      |   | Системный файл инициализации |
| 8  | &rtp/filetype.vim           | Нет      | ''filet on'' | Определение типа файла по имени|
| 7  | &rtp/ftdetect/*.vim         | Нет      | ''filet on'' | Определение типа файла по имени|
| 6  | $HOME/.vimrc                | Нет      |   | Пользовательский файл инициализации |
| -  | rtp=$VIM,$HOMEVIM           | -------- | - | - |
| 7  | &rtp/ftdetect/*.vim         | Нет      | ''filet off | filet on'' | Определение типа файла по имени|
| 8  | &rtp/filetype.vim           | Нет      | ''filet off | filet on'' | Определение типа файла по имени|
| 9  | ./.vimrc                    | Нет      | ''exrc'' | Проектный файл инициализации |
| -  | rtp=$VIM,$HOMEVIM,./.vim    | -------- | - | - |

| 11 | &rtp/filetype.vim           | Нет      | ''filet off | filet on'' | Определение типа файла по имени|

===== Контекстно-зависимые плагины, опции и привязки =====

Контекстно-зависимые плагины, такие как ''ftplugin'', считываются из ФС и исполняются повторно при определении контекста редактором в виде события ''FileType''. Определение контекста выполняется при инициализации буфера.

Контекстно-зависимые опции и привязки (локальные по отношению к буферу) удаляются только при удалении буфера из памяти редактора. Переход между буферами не удаляет эти данные.
