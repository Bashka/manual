====== Метки ======

Метки в Git это указатели на некоторый комит, с помощью которых можно удобно ориентироваться в истории. Существует два типа метод:

  - Легковесные - именованная ссылка на комит
  - Аннотированные - полноценный объект-ссылка с описанием, автором и датой

===== Легковесные метки =====

Для создания легковесной метки используется команда ''%%git tag метка%%''. Эта команда помечает последний комит. Если же необходимо пометить определенный комит, используется команда ''%%git tag метка комит%%''.

Пример:

  git tag v0.8

===== Аннотированные метки =====

Для создания аннотированной метки используется команда ''%%git tag -a метка%%'', которая требует ввести описание метки в текстовом редакторе. Эта команда помечает последний комит. Если же необходимо пометить определенный комит, используется команда ''%%git tag -a метка комит%%''.

Команда имеет следующие ключи:

  * ''%%git tag -m "описание" ...%%'' - добавления описания метки при вызове команды
  * ''%%git tag --file=файл ...%%'' - получение описания метки из файла
  * ''%%git tag -l -n<число>%%'' - вывод вместе с метками их описания. Число определяет количество строк, которые будут выведены

====== Список меток ======

Посмотреть весь список созданных в репозитории меток можно с помощью команды ''%%git tag%%''.

Если необходимо получить только метки, соответствующие шаблону, используется команда ''%%git tag -l 'шаблон'%%''.

====== Информация о метке ======

Для получения подробной информации о метке, используется команда ''%%git show метка%%''. Для аннотированной метки она покажет автора, дату создания, описание и список изменений, для легковесной, только комит, который она помечает.

====== Перемещение к метке ======

Для возврата к состоянию, определенному комитом метки используется команда ''%%git checkout метка%%''.

====== Удаление меток ======

Для удаления меток используется команда ''%%git tag -d метка%%''.
