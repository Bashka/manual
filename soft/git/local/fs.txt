====== Удаление файла ======

Обычное удаление файла, находящегося под версионным контролем, не позволяет проиндексировать изменения путем указания имени файла или группы файлов:

  $>rm deleteFile.txt
  $>git add deleteFile.txt
  $>git status
  # On branch master
  # Changes not staged for commit:
  #
  #	deleted:    new

Это связано с тем, что на момент индексации файла нет в файлойвой системе.

Для индексации таких файлов используется команда ''%%git add -u%%''.

Git позволяет удалить файл с помещением этих изменений в индекс с помощью команды ''git rm файл''.

Команда имеет следующие ключи:

  - ''git rm -f ...'' - принудительное удаление файла
  - ''git rm -r ...'' - рекурсивное удаление директории
  - ''%%git rm --dry-run ...%%'' - файл не удаляется из репозитория и файловой системы ("холостой ход")
  - ''git rm -q ...'' - команда не возвращает информацию о ходе работы ("тихий режим")

===== Исключение из версионного контроля =====

Для удаления файла из под версионного контроля, но не из файловой системы, используется команда ''%%git rm --cached файл%%''.

====== Перемещение и переименование файла ======

Аналогично удалению файла, операция перемещения/переименования должна сопровождаться индексацией этих изменений. Делается это с помощью команды ''git mv файл новоеИмя''.

Команда имеет следующие ключи:

  - ''git mv -f ...'' - принудительное перемещение/переименование
  - ''git mv -v ...'' - команда сообщает дополнительную информацию об изменениях
  - ''%%git mv --dry-run ...%%'' - команда не выполняет реального перемещения/переименования ("холостой ход")
