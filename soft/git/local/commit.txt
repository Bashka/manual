====== Фиксация состояния репозитория ======

Для фиксации изменений в Git используется команда ''git commit''. Она записывает состояние индекса в истории репозитория (комит) так, чтобы к нему можно было вернуться в будущем.

После выполнения данной команды, Git откроет используемый пользователем редактор для того, чтобы можно было ввести описание комита.

Команда имеет следующие ключи:

  * ''%%git commit -a%%'' - автоматическая индексация и комит измененных или удаленных файлов, находящихся под версионным контролем (новые файлы не затрагиваются)
  * ''%%git commit -m "коментарий"%%'' - определение описания комита без открытия редактора

  git commit -m "First commit"

  * ''%%git commit -C комит%%'' - комит через копирование описания и автора из указанного комита
  * ''%%git commit -c комит%%'' - комит через копирование описания и автора из указанного комита с возможностью редактировать описание
  * ''%%git commit --amend%%'' - изменение описания последнего комита (аналог ''%%git reset --soft HEAD^ && git commit -c ORIG_HEAD%%''
  * ''%%git commit --reset-author [-C|-c|--amend]%%'' - аналогично ключам -C, -c или -amend, но без копирования автора
  * ''%%git commit --file=файл%%'' - комит с получением описания из файла
  * ''%%git commit --template=файл%%'' - использование файла в качестве шаблона для описания комита
  * ''%%git commit --status%%'' (по умолчанию) - добавить в описание комита статус репозитория (git status)
  * ''%%git commit --no-status%%'' - не добавлять в описание комита статус репозитория (git status)
  * ''%%git commit --status -uno%%'' - не отображать в статусе репозитория неотслеживаемые файлы
  * ''%%git commit -q%%'' - запретить вывод сообщения о результате работы утилиты ("тихий режим")
  * ''%%git commit -v%%'' - выводить подробную информацию о результатах работы команды
  * ''%%git commit --dry-run%%'' - комит не фиксируется, но утилита возвращает ту же информацию, что при нормальной работе («холостой ход»)
