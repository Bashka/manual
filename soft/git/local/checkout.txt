====== Отмена изменений ======

Если необходимо отменить все несохраненные изменения в проекте, используется команда ''%%git checkout -f HEAD%%''.

Пример:

  # Создание нового файла
  touch newFile.txt
  # Запись данных в существующий файл
  echo 'Hello world' >> file.txt
  # Отмена изменений
  git checkout -f HEAD

====== Перемещение между комитами ======

Для возврата проекта в состояние, записанное в прошлом комите (возврат по истории комитов) используется команда ''%%git checkout комит%%''. Данная команда смещает указатель текущего состояния комита (''HEAD'') на заданный комит. При этом файлы репозитория возвращаются в то состояние, в котором они были на момент этого комита.

Если необходимо вернуться на последний комит, используется команда ''%%git checkout ветка%%'':

  # Перемещение на прошлый комит
  git checkout fb736
  # Возврат к текущему комиту
  git checkout master

Если в проекте имеются несохраненные изменения, команда не будет выполнена. Для принудительного выполнения команды (несохраненные изменения и индекс будут потеряны) используется ключ ''%%git checkout -f комит%%''.

Команда имеет следующие ключи:

  * ''%%git checkout -q ...%%'' - команда не возвращает информацию о результатах работы ("тихий режим")

===== История файлов =====

Если необходимо увидить состояние конкретного файла для данного комита, используется команда ''%%git checkout комит -- адрес%%''.

Аналогично, вернуть файл в состояние, определенное последним комитом можно с помощью команды ''%%git checkout ветка -- адрес%%''.

====== Перемещение между метками ======

Для возврата проекта в состояние, записанное указанной меткой, используется команда ''%%git checkout метка%%'':

  git checkout v1.4
